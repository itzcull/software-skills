# Domain Storytelling in Domain-Driven Design

## Definition

Domain Storytelling is a communication technique in Domain-Driven Design that allows domain experts to illustrate workflows and capture business processes through visual storytelling.

## Key Characteristics

- Uses iconography and arrows to represent:
  - Actors
  - Activities
  - Work objects

## Goals

1. Develop [ubiquitous language](./ubiquitous-language.md)
2. Identify [bounded contexts](./bounded-context.md)
3. Establish requirements and project scope

## Process

Domain storytelling involves collaborative workshops where participants work together to map out business processes through visual narratives.

## Roles

- Moderator/Facilitator
- Modeler
- Domain Experts (Storytellers)
- Listeners

## Workshop Tools

### Physical Workshop
- Sticky notes
- Large paper or poster board
- Drawing tools

### Virtual Workshop
- Digital whiteboard
- Collaborative tools (e.g., Miro)
- Drawing software
- Iconography resources

## Example Scenario

A webinar registration process:

1. User registers on website
2. Website tags user's interest
3. Website sends meeting information
4. User joins online meeting
5. Website sends follow-up email
6. User watches webinar recording

## Benefits

- Creates shared understanding of business processes
- Facilitates communication between technical and business teams
- Helps identify domain boundaries
- Supports requirements gathering
