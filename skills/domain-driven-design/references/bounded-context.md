# Bounded Context

A bounded context is a concept from Domain-Driven Design that establishes a boundary within a domain and contains related models. Key characteristics include:

## Definition
- Contains models, [entities](entity.md), [value objects](value-object.md), and aggregates
- Uses a [ubiquitous language](ubiquitous-language.md) consistently across documentation and communication
- A domain may have multiple bounded contexts

## eCommerce Example Bounded Contexts
Potential bounded contexts in an eCommerce system:
- Catalog
- Content Management
- Customer
- Fraud Detection
- Inventory Management
- Marketing
- Order
- Product Recommendation

## Context Mapping
- Explains relationships between bounded contexts
- Indicates team interactions and dependencies
